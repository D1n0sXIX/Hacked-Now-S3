# Hacked-Now-S3

Proyecto: Hacked-Now-S3 — Planificación y documentación para el Sprint 3

Resumen
-------
Este repositorio contiene la documentación del proyecto y la planificación para el Sprint 3. El objetivo es diseñar y desarrollar una WebApp funcional, preparar la presentación del proyecto y obtener feedback de posibles usuarios/cliente.

Objetivos
---------
- Entregar una WebApp desplegada y funcional.
- Preparar una presentación de 5–10 diapositivas (más backup slides).
- Entrevistar al menos 2 posibles usuarios/clientes para validación.

Cosas que hay que hacer (entregables)
-----------------------------------
- WebApp desplegada y accesible.
- Presentación principal (5–10 diapositivas).
- Backup slides con fuentes, URLs y resumen del sprint.
- Guion de presentación (≈8 minutos).
- Registro de entrevistas con al menos 2 clientes potenciales.

Guion de la presentación
------------------------
1. **Cliente**: Historia que define y perfila al cliente objetivo (diapositivas).
2. **Necesidades**: Tareas y necesidades del cliente (diapositivas).
3. **Problemas**: Problemas actuales y resultados deseados (diapositivas).
4. **Alternativas**: Soluciones existentes y sus limitaciones (diapositivas).
5. **Demo en vivo**: Demostración de la WebApp y por qué nuestra solución es única.

Diseño de la web --> Desarrollada con NEXT
----------------
Páginas principales y comportamiento esperado:

- `Inicio`: Introducción al proyecto, CTA para publicar casos (lleva a `PublicarCaso`), resumen de cómo funciona la web y secciones destacadas.
- `Foro`: Listado de casos, búsqueda y filtros por categoría, opción para publicar nuevo caso.
- `CasoDelForo`: Página de detalle de un caso con comentarios y metadatos.
- `ZonaDeNoticias` / `Noticia`: Listado y vista individual de noticias (cada noticia puede abrir un overlay/modal).
- `PublicarCaso`: Formulario para añadir nuevos casos (categoría, etiquetas, visibilidad: público/experto/anónimo).
- `LogIn`: Inicio de sesión sencillo.
- `PlanDesarrollador`: Página informativa para posibles consultores (no funcional, solo presentación de requisitos).
- `PaginaUser`: Perfil de usuario, ajustes y notificaciones.
- `NavBar`: Navegación global disponible en todas las páginas.
